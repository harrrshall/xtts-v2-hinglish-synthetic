# syntts — Hinglish TTS from synthetic data

Train a Hinglish (Hindi-English code-switched) text-to-speech model on **purely synthetic** data
(generated by the teacher TTS teacher) **without losing the teacher's quality**.

Status: **complete and objectively verified.** A fine-tuned XTTS-v2 student, trained only on
synthetic audio, matches the human-verified teacher within <=3% on every measured axis.

## Result (n=89 held-out paired sentences, bootstrap 95% CI)

| Axis | Student vs teacher | 95% CI |
|------|--------------------|--------|
| Intelligibility (ASR recall) | -1.6% | [-3.3%, 0.0%] |
| Accent (English-as-English)  | -3.0% | [-6.2%, -0.1%] |
| Naturalness (UTMOS)          | +0.10 | [+0.02, +0.18] |
| Voice copy-fidelity (SECS)   | +0.9% | [+0.3%, +1.5%] |

Full write-up with caveats: [`docs/FINAL_REPORT.md`](docs/FINAL_REPORT.md).

## The model

**On Hugging Face:** [`harrrshall/xtts-v2-hinglish-synthetic`](https://huggingface.co/harrrshall/xtts-v2-hinglish-synthetic)
(public, CPML / non-commercial). The model card there has the full synthetic-data recipe, the
training process, and a copy-paste usage snippet. The card source is [`docs/HF_MODEL_CARD.md`](docs/HF_MODEL_CARD.md).

An optional **fp16 build** (`model_fp16.pth`, ~1.04 GB, half the size) ships alongside the fp32
default. It is fp16-on-disk / fp32-at-runtime, verified to have no quality loss three independent
ways (numerical equivalence + greedy-paired + sampled-paired UTMOS/SECS). See
[`docs/FP16_VERIFICATION.md`](docs/FP16_VERIFICATION.md) and [`scripts/hinglish/fp16/`](scripts/hinglish/fp16/).

The trained model and full-scale outputs also live on the GPU box (compute environment), not in this
repo, because they are large:
- Model release: `runs/xtts_hinglish/RELEASE/` (model + config + vocab +
  speakers + per-voice reference clips + INFERENCE.md)
- Full synth corpus, filter scores, training run: under `data` and `/runs`

This repo holds the code, the docs, the real-data inputs, and the pulled-back evaluation evidence
and sample clips.

## Run it

The quickest way to use the model is from Hugging Face (no repo setup needed); see the
[model card](https://huggingface.co/harrrshall/xtts-v2-hinglish-synthetic) for a copy-paste snippet.

To run inference from this repo on a CUDA machine:
```
python scripts/hinglish/07_xtts_infer.py
```
Generates Hinglish speech in the 4 fixed voices. Operational notes: spell numbers as words (not
digits), keep text under ~150 chars per call. Full pipeline order is in [`docs/RUNBOOK.md`](docs/RUNBOOK.md).

## Pipeline at a glance

corpus text (real seed + generated, 2-gate verified) -> teacher TTS synthesis (5,880 clips) ->
qwen3-asr bin-aware filter -> training manifest (2,945 clips) -> XTTS-v2 fine-tune (8 epochs) ->
objective eval vs teacher.

## Compute

- **GPU:** NVIDIA H200 NVL (143 GB). GPU-side stages used up to 3 cards in parallel.
- **GPU hours (approximate, ~2-3 H200-hours total):**
  - Fine-tuning: ~0.2 h on 1x H200 (XTTS-v2 GPT, 8 epochs, ~2,855 clips).
  - Filtering: ~1 h of qwen3-asr inference over the 5,880-clip synth set, sharded across 3x H200.
  - Eval + probes: ~1 h (UTMOS / speaker-sim / ASR recall / accent ASR, the smoke test, and the
    teacher-gate round-trip).
- Note: teacher synthesis (5,880 clips) ran against a hosted TTS API, so it is network/CPU time,
  not GPU time.
